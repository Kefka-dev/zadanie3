Špecifikácia zadania


Cieľom zadania je napísať program na editáciu a vyhľadávanie informácií v štruktúrovaných konfiguračných súboroch (inšpiráciu sme čerpali z formátu konfiguračných INI súborov). Po skompilovaní sa program spustí v termináli spolu s podporovanými prepínačmi (command-line argumenty). Jednotlivé prepínače určujú režim (vyhľadávací/editačný), v ktorom bude program pracovať. Konfiguračné súbory sú zadané do programu ako riadky textu z klávesnice (štandardný vstup). Vzhľadom na náročnosť/zdĺhavosť zadávania textu cez klávesnicu, odporúčame použiť presmerovanie štandardného vstupu z textového súboru. Po načítaní vstupného textu sa vyhľadajú alebo aktualizujú záznamy vyhovujúce zadaným filtrom. Výsledkom je vždy vypísanie údajov (vyhľadaných/aktualizovaných) konfiguračného súboru v požadovanom formáte na štandardný výstup. Okrem štandardného behu programu, treba ošetriť predpísané chybové situácie, na ktoré program reaguje ukončením jeho činnosti a výpisom príslušného chybového hlásenia. Zmyslom zadania je naučiť sa pracovať s command-line argumentmi, načítavať a spracovávať textové reťazce.

Obrázok 1: Fázy činnosti programu

CMD argumenty

CMD argumenty sú argumenty programu, ktoré sa zadávajú v termináli pri jeho spustení. Slúžia na ovládanie správania sa programu z miesta jeho spustenia. Môžeme zadať ľubovoľný počet argumentov v ľubovoľnom tvare (okrem niektorých špeciálnych znakov, ktoré sú osobitne interpretované terminálom). Každý CMD argument je reprezentovaný ako C reťazec. Všetky CMD argumenty sú k dispozícii prostredníctvom poľa reťazcov argv, ktoré má dĺžku argc prvkov. Prvok argv[0] obsahuje cestu k spúšťanému programu. Ostatné prvky poľa argv predstavujú používateľom zadané CMD argumenty. Pole argv a jeho dĺžka argc sú parametrami hlavnej funkcie main.

Príklad spustenia programu s CMD argumentmi v Linux termináli:

./program 123 abc "hello world"

Daný príklad ilustruje situáciu spustenia skompilovaného programu program v termináli so zadaním CMD argumentov. Jednotlivé argumenty sú vymenované v nasledovnom zozname.

  • Prvok argv[0] obsahuje cestu k programu program.
  • Prvok argv[1] obsahuje reťazec "123". Pozor, jedná sa o textovú reprezentáciu čísla.
  • Prvok argv[2] obsahuje reťazec "abc".
  • Prvok argv[3] obsahuje reťazec "hello world". Aj napriek tomu, že tento reťazec pozostáva z viacerých slov oddelených medzerou, je stále chápaný ako samostatný argument, nakoľko je uzatvorený v úvodzovkách.

V zdrojovom kóde vieme pristúpiť k CMD argumentom pomocou parametrov funkcie main. 

int main(int argc, char* argv[])
{
    /*
        argc - pocet CMD argumentov
        argv - pole argumentov (technicky je to pole retazcov)
    */
    return 0;
}

V tomto zadaní budeme rozlišovať tieto 3 typy CMD argumentov:

  • Prepínače (z angl. options). Prepínačom rozumieme CMD argument v tvare -x, kde x je ľubovoľné písmeno. Prepínače budú reprezentovať príslušné operácie spracovania načítaného textu.
  • Argumenty prepínačov. Prepínač môže mať stanovený aj svoj argument, napr. -p abc, kde za prepínačom -p nasleduje jeho argument abc.
  • Non-option argumenty. Každý CMD argument, ktorý nepatrí ani do jednej z dvoch vyššie uvedených kategórií je považovaný za tzv. non-option argument. V našom prípade budú všetky non-option argumenty umiestnené až za všetkými prepínačmi a ich argumentmi.

Treba sa vysporiadať aj so situáciami, kedy používateľ zadá neplatný prepínač alebo zabudne uviesť povinný argument platného prepínača. Navyše, okrem prepínačov a ich argumentov vieme program spustiť aj so zvyšnými non-option argumentmi.

Ukážka rôznych typov CMD argumentov:

./program -a -b param arg1 arg2 arg3

Popis zadaných CMD argumentov:

  • -a je prepínač bez argumentu
  • -b je prepínač s argumentom param
  • arg1, arg2 a arg3 sú non-option argumenty

Funkcia getopt

Proces rozpoznania prepínačov, ich argumentov a non-option argumentov sa dá jednoducho automatizovať pomocou knižničnej funkcie getopt. Funkcia getopt okrem iného odhaľuje aj situácie nesprávneho použitia prepínačov. Odporúčame preto využiť túto funkciu, ušetrí vám veľa nervov a času. Funkciu getopt je možné použiť len v Unix-like systémoch. Používatelia Windows musia mať preto na jej využitie nainštalované vývojové prostredie MinGW, Cygwin, MSYS2 alebo WSL a používať ním distribuovaný kompilátor GCC. Funkciu getopt je možné použiť po vložení hlavičkového súboru unistd.h.

Hlavička funkcie getopt:

int getopt(int argc, char *const argv[], const char *optstring);
  • argc je počet CMD argumentov
  • argv je pole CMD argumentov (každý argument je reťazec)
  • optstring je reťazec špecifikujúci platné prepínače (t.j. také, ktoré váš program rozpoznáva) a taktiež uvádza, či dané prepínače majú argumenty

Nasledujúci príklad demonštruje použitie funkcie getopt pri spracovaní CMD argumentov programu.

#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>

int main(int argc, char *argv[]){

	// Poznamky ku 'getopt':

	// Retazec 'optstring' specifikuje platne prepinace programu.

	// Ak funkcia 'getopt' narazi na neplatny prepinac, vrati znak '?'.
	// Ak 'optstring' zacina znakom ':', vtedy 'getopt' vrati znak ':'
	// v pripade chybajuceho povinneho argumentu prepinaca.

	// Ak je v 'optstringu' za pismenom prepinaca znak ':', znamena to,
	// ze dany prepinac ma povinny argument.

	// Povinny argument aktualneho prepinaca vieme ziskat cez globalny
	// smernik 'optarg'.

	// Ked su rozpoznane vsetky prepinace, funkcia 'getopt' vrati -1.
	// Ak zostali nejake non-option argumenty, tak prvy non-option argument
	// ma index dany globalnou premennou 'optind'.

	int opt;
	char* optstring = ":ab:c";

    while ((opt = getopt(argc, argv, optstring)) != -1) {
        switch (opt) {
            case 'a':
                printf("Prepinac -a\n");
                break;
            case 'b':
                printf("Prepinac -b a jeho povinny argument %s\n",optarg);
                break;
            case 'c':
            	printf("Prepinac -c\n");
            	break;
            default: /* '?' alebo ':' */
                printf("Neplatny prepinac alebo chybajuci argument prepinaca\n");
                return 1;
        }
    }
    if(optind < argc){
    	for(int i=optind;i<argc;i++){
    		printf("Non-option CMD argument: %s\n",argv[i]);
    	}
    }

	return 0;
}

Skompilovaný program spustíme z terminálu v tvare:

./program -a -b ahoj -c slovo1 slovo2

Dostaneme nasledovný výstup:

Prepinac -a
Prepinac -b a jeho povinny argument ahoj
Prepinac -c
Non-option CMD argument: slovo1
Non-option CMD argument: slovo2

Vstup

Program po spustení načíta z klávesnice (štandardný vstup) vstupný text. Vstupný text bude predstavovať štruktúrovaný konfiguračný súbor, ktorý bude program spracovávať. Načítavanie textu končí v prípade zadania 2 po sebe idúcich prázdnych riadkov (prázdny riadok je riadok obsahujúci iba znak '\n'). Tieto posledné 2 prázdne riadky nie sú súčasťou načítaného textu.

Štruktúra načítaného konfiguračného súboru

Po spustení začne používateľ zadávať riadky textu. Tieto riadky reprezentujú konfiguračný súbor, ktorý má pevne danú štruktúru. Maximálna dĺžka riadku konfiguračného súboru je 1000 znakov. Konfiguračný súbor sa skladá z nasledovných častí:

  • Sekcia

    Konfiguračný súbor je rozdelený do sekcií. Začiatok sekcie je reprezentovaný riadkom v tvare [section_name], kde section_name predstavuje názov danej sekcie. Susedné sekcie sú od seba oddelené najviac 1 prázdnym riadkom. Názvy sekcií budú obsahovať len malé/veľké písmená abecedy, číslice alebo medzery. Názvy sekcií sú unikátne. Sekcie môžu byť aj prázdne, t.j. nemusia obsahovať žiadne dvojice kľúč=hodnota.

  • Dvojica kľúč=hodnota

    V každej sekcii je ľubovoľný počet riadkov (0 a viac), ktoré sú v tvare kľúč=hodnota. Dvojice kľúč=hodnota nasledujú bezprostredne za riadkom s názvom sekcie. Oddeľovačom kľúča a hodnoty je znak =. Kľúč aj hodnota predstavujú ľubovoľný reťazec, pre ktorý platia nasledujúce obmedzenia. Kľúče a hodnoty môžu obsahovať len malé/veľké písmená abecedy, číslice, interpunkciu a medzery. Kľúče a hodnoty nikdy nebudú obsahovať znaky =, [ a ]. Všetky kľúče v sekcii sú unikátne. Každý kľúč musí mať hodnotu.

    Dôležitá poznámka k načítavaniu kľúčov a hodnôt:

    Kľúč začína prvým znakom v riadku, ktorý nie je bielym znakom (všetko okrem medzier, tabulátorov a znakov nového riadku). Kľúč čítame zľava doprava až pokiaľ nenarazíme na znak, ktorý nie je biely a zároveň za ním nasleduje už len ľubovoľný počet bielych znakov a znak oddeľovača =.

    Hodnotu kľúča začíname hľadať od znaku oddeľovača =. Hodnota kľúča začína prvým znakom, ktorý nie je bielym znakom. Hodnotu kľúča čítame zľava doprava pokiaľ nenarazíme na znak, ktorý nie je biely a zároveň za ním nasleduje už len ľubovoľný počet bielych znakov a znak nového riadku.

Presmerovanie štandardného vstupu

Zadávanie vstupného textu po riadkoch z klávesnice môže byť nekomfortné a zdĺhavé. Preto odporúčame pri testovaní vašich programov presmerovať štandardný vstup z textového súboru. V textovom súbore bude uložený štruktúrovaný text predstavujúci konfiguračný súbor. Majme skompilovaný program s názvom z3 a konfiguračný súbor config.txt. Program s presmerovaným štandardným vstupom (zo súboru config.txt) spustíme z terminálu nasledovným príkazom.

./z3 < config.txt

Chybové situácie

V programe môžu nastať tieto 3 chybové situácie:

  • E1

    Ak je program v termináli spustený s neznámym/neplatným prepínačom. Program vtedy vypíše chybové hlásenie E1 a korektne ukončí svoju činnosť (hlavná funkcia main vráti hodnotu 0). Iné výpisy nie sú tolerované.

    Príklady situácie E1: 
    ./z3 -p
./z3 -p -s Audio

    Poznámka: v tomto prípade bol zadaný neplatný prepínač -p.

  • E2

    Ak je program v termináli spustený s chýbajúcim povinným argumentom prepínača. Program vtedy vypíše chybové hlásenie E2 a korektne ukončí svoju činnosť (hlavná funkcia main vráti hodnotu 0). Iné výpisy nie sú tolerované.

    Príklady situácie E2: 
    ./z3 -k
./z3 -s

    Poznámka: v tomto prípade neboli zadané povinné argumenty prepínačov -k a -s.

  • E3

    Ak je program v termináli spustený s prepínačom -u a nebol zadaný žiadny non-option argument alebo bol zadaný viac ako 1 non-option argument. Program vtedy vypíše chybové hlásenie E3 a korektne ukončí svoju činnosť (hlavná funkcia main vráti hodnotu 0). Iné výpisy nie sú tolerované.

    Príklady situácie E3: 
    ./z3 -u Audio.Volume
./z3 -u Description

    Poznámka: v tomto prípade nebol zadaný žiadny non-option argument, ktorý je požadovaný v tomto režime činnosti.

Funkcionalita

Rozlišujeme 4 režimy činnosti programu (podľa zadaného prepínača):

  • Spustenie programu bez prepínača
  • Prepínač -k: vyhľadávanie hodnôt kľúčov
  • Prepínač -s: vyhľadávanie kľúčov
  • Prepínač -u: aktualizácia/pridanie novej dvojice kľúč=hodnota
Spustenie programu bez prepínača

Ak program spustíme bez prepínača, vypíše sa celý načítaný konfiguračný súbor v pôvodnej podobe.

Prepínač -k: vyhľadávanie hodnôt kľúčov

V tomto režime sa vyhľadávajú hodnoty kľúčov. Vypíšu sa len tie hodnoty, ktoré sa nachádzajú v zadanej sekcii a prislúchajú zadanému kľúču. Vyhovujúce hodnoty kľúčov sa vypíšu na štandardný výstup.

Spustenie programu:

./z3 -k [section.]key

Ak je program spustený v tomto režime, očakáva sa, že používateľ poskytne povinný argument prepínača -k, ktorý má tvar [section.]key. Povinný argument sa skladá z 2 častí:

  • [section.]

    Nepovinná časť argumentu, ktorá špecifikuje názov sekcie section, v ktorej sa má hodnota kľúča vyhľadávať. Ak je zadaný názov sekcie, nasleduje separátor . (bodka), za ktorým nasleduje názov kľúča.

  • key

    Časť argumentu, ktorá špecifikuje názov kľúča key, ktorého hodnotu hľadáme.

V tomto režime činnosti môžu nastať nasledovné situácie:

  • Sekcia sectionje zadaná a existuje.

    Program vtedy vypíše hodnotu zadaného kľúča v danej sekcii. Keďže sekcie a kľúče sú unikátne, vypíše sa len jeden riadok. Ak zadaný kľúč v sekcii neexistuje, program nič nevypíše.

  • Sekcia section je zadaná a neexistuje.

    Program nič nevypíše.

  • Sekcia nie je zadaná.

    Program vypíše hodnoty zadaného kľúča vo všetkých sekciách. Každá hodnota kľúča je vypísaná na samostatnom riadku. Ak sa zadaný kľúč v konfiguračnom súbore nenachádza v žiadnej sekcii, program nič nevypíše.

Prepínač -s: vyhľadávanie kľúčov

V tomto režime sa vyhľadávajú kľúče. Vypíšu sa len tie kľúče, ktoré sa nachádzajú v zadanej sekcii. Vyhovujúce kľúče sa vypíšu na štandardný výstup.

Spustenie programu:

./z3 -s section

Ak je program spustený v tomto režime, očakáva sa, že používateľ poskytne povinný argument prepínača -s, ktorý reprezentuje názov sekcie section, v ktorej sa vyhľadáva. Ak sekcia section neexistuje, program nič nevypíše. Ak je namiesto názvu sekcie zadaný reťazec "*", program vypíše kľúče vo všetkých sekciách.

Formát výpisu kľúčov:

Kľúče sa vypisujú na samostatných riadkoch vo formáte section.key, kde section predstavuje názov sekcie a key predstavuje kľúč. V prípade výpisu kľúčov vo všetkých sekciách (reťazec "*" je zadaný ako argument prepínača -s) sa kľúče rôznych sekcií neoddeľujú prázdnym riadkom.

Prepínač -u: aktualizácia/pridanie novej dvojice kľúč=hodnota

V tomto režime sa aktualizujú/pridávajú dvojice kľúč=hodnota. Vždy sa na štandardný výstup vypíše celý konfiguračný súbor vrátane aktualizovaných/pridaných informácií.

Spustenie programu:

./z3 -u [section.]key arg

Ak je program spustený v tomto režime, očakáva sa, že používateľ poskytne povinný argument prepínača -u, ktorý má tvar [section.]key. Povinný argument sa skladá z 2 častí:

  • [section.]

    Nepovinná časť argumentu, ktorá špecifikuje názov sekcie section, v ktorej sa má aktualizovať hodnota zadaného kľúča alebo do ktorej sa má pridať nová dvojica kľúč=hodnota. Ak je zadaný názov sekcie, nasleduje separátor . (bodka), za ktorým nasleduje názov kľúča.

  • key

    Časť argumentu, ktorá špecifikuje názov kľúča key, ktorého hodnotu ideme aktualizovať resp. ktorý ideme pridať.

Okrem povinného argumentu prepínača -u sa očakáva, že program bude spustený s jedným non-option argumentom arg, ktorý bude uvedený na konci všetkých CMD argumentov programu. Hodnota arg predstavuje hodnotu kľúča key.

V tomto režime činnosti môžu nastať nasledovné situácie:

  • Sekcia sectionje zadaná a existuje, kľúč key existuje.

    Program vtedy aktualizuje hodnotu kľúča key v sekcii section na novú hodnotu arg.

  • Sekcia section je zadaná a existuje, kľúč key neexistuje.

    Program vtedy pridá do sekcie section novú dvojicu kľúč=hodnota, kde kľúčom bude key a jeho hodnotou bude arg.

  • Sekcia section je zadaná a neexistuje.

    Program vtedy vytvorí novú prázdnu sekciu s názvom section a pridá do nej dvojicu kľúč=hodnota, kde kľúčom bude key a jeho hodnota bude arg. Takto vzniknutá sekcia bude pridaná na koniec konfiguračného súboru. Pred sekciou sa vypíše 1 prázdny riadok.

  • Sekcia section nie je zadaná, kľúč key existuje.

    Program vtedy aktualizuje hodnotu kľúča key vo všetkých sekciách na novú hodnotu arg.

  • Sekcia section nie je zadaná, kľúč key neexistuje.

    Program vtedy vytvorí novú prázdnu sekciu s predvoleným názvom "My section" a pridá do nej dvojicu kľúč=hodnota, kde kľúčom bude key a jeho hodnota bude arg. Takto vzniknutá sekcia bude pridaná na koniec konfiguračného súboru. Pred sekciou sa vypíše 1 prázdny riadok.
